PSO-ACO Hybrid Dynamic Path Planner for Dhaka City — Based on index.html Implementation
1️⃣ PSO Optimization (Parameter Tuning)
Particle Swarm Optimization to find optimal α, β, ρ for ACO
Key Formulas Used (from code — PSOOptimizer class):
v[d] = 0.72 × v[d] + 1.49 × r₁ × (pBest[d] - pos[d]) + 1.49 × r₂ × (gBest[d] - pos[d])
pos[d] = pos[d] + v[d]
Bounds: α ∈ [0.5, 5.0], β ∈ [0.5, 5.0], ρ ∈ [0.05, 0.95]
fitness(α,β,ρ) = bestPathCost from ACO(10 ants, 15 iters, useRoadFactor=true)

2️⃣ Improved ACO (Road Condition Factor)
Ant Colony Optimization using R_i(t) = d_i × (1 + C_i) as edge cost
Key Formulas (ACOPathfinder class, useRoadFactor = true):
C_i(t) = (vehicleCount × 4.5) / (lanes × distance)
R_i(t) = d_i × (1 + C_i(t)) ← ROAD CONDITION FACTOR
η_ij = 1 / R_i(t) ← heuristic uses R, not d
P_ij = [τ_ij]^α × [η_ij]^β / Σ ← transition probability
Evaporation: τ = τ × (1 − ρ), min = 0.001
Deposit: Δτ = Q / pathCost
Elitist: Δτ_best = (Q / bestCost) × 2
3️⃣ Standard ACO (Distance-Based Baseline)
Same ACO algorithm but using physical distance d_i instead of road factor R_i
Comparison Metrics (Metrics class):
Standard: edgeCost = d_i (distance in meters)
Improved: edgeCost = R_i(t) = d_i × (1 + C_i(t))

CR = Σ(R_i for congested edges) / Σ(R_i for all edges)
where "congested" = C_i > 0.3

CR_promote = (CR_standard − CR_improved) / CR_standard × 100%
Positive value = improved method avoids more congested roads ✅
4️⃣ Dynamic Simulation (Real-time Rerouting)
Vehicle moves along path, traffic updates each tick, reroutes when congestion exceeds thresholds
Key Dynamic Rerouting Logic (DynamicPlanner class):
Vehicle speed on edge: actualSpeed = speed × 1/(1 + C_i)
→ Higher congestion = slower movement (realistic!)

REROUTE TRIGGER CONDITIONS (checked every 30 frames):
  Condition 1: ΔR_i = |R_i(t) − R_i(t−1)| > ΔR_max (sudden change)
  Condition 2: R_i(t) > R_max (absolute congestion limit)

If triggered: Run new ACO from currentNode → destination
  Using PSO-optimized params, 15 ants, 30 iterations
  New path replaces old path, reroute counter increments

Traffic Model: flow = base + morning_peak×gauss(h,8.5,1.0) + evening_peak×gauss(h,18,1.5)
